## 三、架构设计

### 3.1 核心理念

**工作台 = 展示层，主数据 = 文件层**

- 工作台只负责展示和操作，不存储业务数据
- 所有数据以 Markdown 文件形式存储在 Obsidian vault 中
- 通过 frontmatter 元数据驱动查询和统计
- 配置驱动，所有可变行为都通过 config.json 控制

### 3.2 分层架构

```
┌─────────────────────────────────────────────┐
│  main.ts（入口层）                           │
│  加载配置 → 注册组件 → 懒初始化              │
├─────────────────────────────────────────────┤
│  views/（视图层）                            │
│  WorkspaceView → StatusBar + QuickActions   │
│                → ActionTab + KnowledgeTab   │
│                → AnalysisTab + ReadingTab   │
├─────────────────────────────────────────────┤
│  services/（服务层）                         │
│  PlanService + WorkflowService              │
│  InitService + PropertyService              │
├─────────────────────────────────────────────┤
│  config/（配置层）                           │
│  types → defaults → loader → accessors      │
├─────────────────────────────────────────────┤
│  utils/（工具层）                            │
│  dataview（数据查询）+ templates（模板）     │
│  theme（主题）                               │
├─────────────────────────────────────────────┤
│  Obsidian API（平台层）                      │
│  Vault, Workspace, MetadataCache, Events    │
└─────────────────────────────────────────────┘
```

### 3.3 模块依赖关系

```
main.ts
  ├── config/loader.ts        (加载配置)
  ├── config/accessors.ts     (读取配置值)
  ├── services/InitService.ts  (初始化目录)
  ├── services/PropertyService.ts (注册属性)
  ├── services/PlanService.ts  (日/周计划)
  ├── settings/PluginSettings.ts (设置面板)
  ├── views/WorkspaceView.ts   (主视图)
  └── modals/CreateNoteModal.ts (创建事项)

WorkspaceView
  ├── views/StatusBar.ts
  ├── views/QuickActions.ts
  └── views/tabs/*.ts

QuickActions → WorkflowService → dataview.ts + accessors.ts
StatusBar → dataview.ts + templates.ts + accessors.ts
All tabs → dataview.ts + accessors.ts
```

## 四、模块详解

### 4.1 入口层 — main.ts

**职责**：插件生命周期管理

```
onload()
  ├── loadConfig()           → 加载配置
  ├── addSettingTab()        → 注册设置页面
  ├── registerView()         → 注册工作台视图
  ├── addRibbonIcon("layout-grid") → 注册侧边栏图标（首次点击时延迟初始化）
  └── addCommand() × 4       → 注册 4 个命令
```

**延迟初始化**：首次点击侧边栏图标时调用 `ensureInitialized()`，完成以下操作：
- `initWorkspace()` — 初始化目录和示例文件
- `registerPropertyTypes()` — 注册属性类型到 types.json
- `registerEditorSuggests()` — 注册编辑器自动补全

**关键方法**：
- `ensureInitialized()` — 确保已完成首次初始化（懒加载）
- `activateView()` — 激活工作台视图（主编辑区或侧边栏）
- `saveConfig()` — 持久化配置到 config.json
- `refreshWorkspaceView()` — 刷新工作台视图

### 4.2 配置层 — src/config/

#### types.ts — 类型定义

```
AppConfig
  ├── version: number
  ├── basic: BasicConfig
  │   ├── app: { name, icon, viewTitle }
  │   ├── ui: { openInMainEditor, themeId }
  │   ├── folders: { root, structure: Record<string, FolderConfig> }
  │   ├── domains: DomainConfig[]
  │   ├── statuses: { topic[], learning[], default, completed[] }
  │   ├── types: TypeConfig[]
  │   └── workflows: WorkflowConfig[]
  └── advanced: AdvancedConfig
      ├── tabs: TabConfig[]
      ├── fields: Record<string, string>
      ├── limits: Record<string, number>
      ├── theme: { defaultId, statusColors }
      └── templates: { weeklyCategories[] }
```

#### defaults.ts — 默认配置

提供 `DEFAULT_CONFIG` 对象，包含所有配置项的默认值。新用户首次使用时以此为基础。

#### loader.ts — 配置加载器

- `loadConfig(plugin)` — 从 config.json 加载配置，与 DEFAULT_CONFIG 深度合并
- `saveConfigFile(plugin, config)` — 保存配置到 config.json
- `validateConfig(config)` — 验证配置必填字段

#### accessors.ts — 配置访问器

提供 20+ 个便捷函数，所有函数以 `App` 为第一个参数：

```typescript
getConfig(app)              // 获取完整配置
getAppName(app)             // 获取应用名称
getRootFolder(app)          // 获取根目录
getFolderPath(app, key)     // 获取文件夹路径（支持绝对/相对路径）
generateFilePath(app, key)  // 生成文件路径（含模式替换）
getDomains(app)             // 获取领域列表
getStatuses(app)            // 获取状态列表
getTypes(app)               // 获取类型列表
getWorkflows(app)           // 获取工作流列表
getTabs(app)                // 获取标签页列表
getFieldName(app, key)      // 获取字段名
getLimit(app, key)          // 获取数值限制
getStatusColor(app, key)    // 获取状态颜色
// ... 更多
```

**路径拼接逻辑**：
```typescript
function getFolderPath(app, folderKey) {
  const folder = config.basic.folders.structure[folderKey];
  // name 包含 "/" → 绝对路径，直接返回
  if (folder.name.includes("/")) return folder.name;
  // 否则拼接 root
  return `${config.basic.folders.root}/${folder.name}`;
}
```

### 4.3 服务层 — src/services/

#### PlanService.ts — 计划服务

| 函数 | 说明 |
|------|------|
| `createDailyPlan(app)` | 创建今日日计划 |
| `createWeeklyPlan(app)` | 创建本周计划 |
| `addDailyPlanLinkToWeeklyPlan(app, dailyPath)` | 在周计划中添加日计划链接 |

**日计划创建流程**：
1. 计算日计划路径（通过 templates.ts）
2. 按需创建文件夹
3. 检查是否已存在，存在则打开
4. 检查周计划是否存在，不存在则询问
5. 生成日计划内容（通过 templates.ts）
6. 创建文件并打开
7. 关联周计划

#### WorkflowService.ts — 工作流服务

| 函数 | 说明 |
|------|------|
| `generateTaskOverview(app)` | 生成任务总览文档 |
| `generateInboxList(app)` | 生成待整理清单 |
| `generateTaskEval(app)` | 生成任务评估文档 |
| `generateWritingMgmt(app)` | 生成写作管理文档 |
| `generateLearningStatsDoc(app)` | 生成学习统计文档 |
| `generateRecentActivity(app)` | 生成最近动态文档 |

每个函数：查询数据 → 生成 Markdown → 保存到清单文件夹 → 打开文档。

#### InitService.ts — 初始化服务

`initWorkspace(app)` — 首次点击图标时调用：
- 遍历配置中的文件夹结构
- 为空文件夹创建 demo 文件（模板内容）
- 为领域和模块创建示例学习笔记

#### PropertyService.ts — 属性服务

| 函数 | 说明 | 状态 |
|------|------|------|
| `registerPropertyTypes(app, config)` | 写入 .obsidian/types.json，注册属性类型 | 已实现未测试 |
| `registerEditorSuggests(plugin, config)` | 注册编辑器自动补全 | 已实现未测试 |

**工作原理**：
- 将 status、学习状态、type 字段注册为 Obsidian 的 select 属性类型
- 在源码模式下，输入字段名后自动弹出下拉补全

### 4.4 视图层 — src/views/

#### WorkspaceView.ts — 主视图容器

继承 `ItemView`，是整个插件的 UI 容器。

**职责**：
- 初始化容器和主题
- 渲染子组件（StatusBar, QuickActions, 标签页）
- 监听 vault 事件（modify/create/delete/rename）
- 防抖自动刷新（500ms）

#### StatusBar.ts — 状态栏

显示 6 个统计卡片：今日日计划、今日待办、本周待办、逾期、已完成、进行中。

**数据来源**：通过 `dataview.ts` 查询 vault 中的 markdown 文件和 frontmatter。

**点击行为**：生成对应的清单文档并打开。

#### QuickActions.ts — 快捷操作栏

- **快速创建按钮**：从配置读取类型列表，点击后打开 CreateNoteModal
- **工作流按钮**：从配置读取工作流列表，点击后调用 WorkflowService

#### 标签页

| 标签页 | 文件 | 主要数据源 |
|--------|------|-----------|
| 行动指北 | ActionTab.ts | 日计划、周计划、事项、常用入口.md |
| 知识管理 | KnowledgeTab.ts | 事项（按状态筛选）、最近文件 |
| 学习分析 | AnalysisTab.ts | 事项（按学习状态/领域筛选）、活跃度数据 |
| 稍后阅读 | ReadingTab.ts | 待整理文件夹中的文件 |

### 4.5 工具层 — src/utils/

#### dataview.ts — 数据查询层

**核心接口**：
- `TaskItem` — 事项数据结构
- `TopicItem` — 事项详情
- `DomainModuleStatsResult` — 领域模块统计

**查询函数**：

| 函数 | 返回 | 说明 |
|------|------|------|
| `getRecentFiles(app, limit)` | 文件列表 | 最近修改的文件 |
| `getTopicsByStatus(app, status)` | 事项列表 | 按状态筛选事项 |
| `getTopicsByType(app, type)` | 事项列表 | 按类型筛选事项 |
| `getAllTopics(app)` | 事项列表 | 所有事项 |
| `getOverdueTopics(app)` | 事项列表 | 逾期事项 |
| `getLearningStats(app)` | 统计对象 | 学习状态分布 |
| `getActivityData(app)` | 日期→计数映射 | 热力图数据 |
| `getDomainModuleStats(app)` | 领域模块统计 | 按领域/模块统计 |
| `getFolderStats(app)` | 文件夹统计 | 各文件夹文档数量 |

**工具函数**：
- `fileExists(app, path)` — 检查文件/文件夹是否存在
- `ensureFolder(app, path)` — 按需创建文件夹（支持多级）
- `toLocalDateStr(date)` — 日期转本地字符串

#### templates.ts — 模板生成

| 函数 | 说明 |
|------|------|
| `getDailyPlanTemplate(app)` | 生成日计划模板内容 |
| `getWeeklyPlanTemplate(app)` | 生成周计划模板内容 |
| `getTopicTemplate(app, title, type, status)` | 生成事项模板内容 |
| `getDiaryTemplate(app)` | 生成日记模板内容 |
| `getDailyPlanPath(app)` | 计算日计划文件路径 |
| `getWeeklyPlanPath(app)` | 计算周计划文件路径 |

#### theme.ts — 主题工具

- `detectTheme()` — 检测当前是暗色还是亮色模式
- `applyTheme(colors)` — 将主题颜色注入 CSS 变量
- `getThemeColors(themeId, mode)` — 获取指定主题的颜色

### 4.6 设置层 — src/settings/

#### PluginSettings.ts — 设置界面

继承 `PluginSettingTab`，分为基础配置和高级配置两个视图。

**基础配置**：
- 打开方式（主编辑区/侧边栏）
- 主题颜色选择（10 种主题）
- 文件夹名称配置（根目录/日计划/周计划/事项/待整理/节点/清单）
- 状态配置（事项状态/学习状态，支持增删改）
- 类型配置（支持增删改）
- 领域与模块编辑（DomainEditModal，ID 自动生成）

**高级配置**：
- Frontmatter 字段名映射
- 工作流文档名称模式
- 数值限制（最近文件数、搜索结果数等）
- 配置导出/重置

**内部类**：
- `DomainEditModal` — 领域编辑弹窗，支持添加/编辑/删除领域和模块
- `autoId()` — 生成 8 位随机 ID

## 五、数据流

### 5.1 读取数据

```
Obsidian Vault
  ├── Markdown Files
  │   └── Frontmatter (status, type, deadline, ...)
  └── MetadataCache
        ↓
  dataview.ts (查询层)
  ├── getTopicsByStatus()
  ├── getActivityData()
  ├── getDomainModuleStats()
  └── ...
        ↓
  views/ (展示层)
  ├── StatusBar.render()
  ├── ActionTab.render()
  └── ...
```

### 5.2 创建文件

```
用户操作（点击按钮/命令）
  ↓
services/ (服务层)
  ├── PlanService.createDailyPlan()
  ├── WorkflowService.generateTaskOverview()
  └── ...
  ↓
utils/templates.ts (生成内容)
  ├── getDailyPlanTemplate()
  └── ...
  ↓
Obsidian API (写入文件)
  ├── app.vault.create(path, content)
  └── app.workspace.openLinkText(path)
```

### 5.3 实时刷新

```
Vault 事件 (modify/create/delete/rename)
  ↓
WorkspaceView.registerFileWatcher()
  ↓
debounceRefresh() (500ms 防抖)
  ↓
重新渲染所有组件
```

## 六、配置系统

### 6.1 配置文件

**位置**：`.obsidian/plugins/worktop/config.json`

**加载流程**：
1. 读取 config.json（如果存在）
2. 与 DEFAULT_CONFIG 深度合并
3. 验证必填字段
4. 返回 AppConfig 对象

**深度合并规则**：
- 对象类型：递归合并
- 数组类型：直接替换
- 基本类型：用户值覆盖默认值

### 6.2 配置分层

| 层级 | 说明 | 用户操作频率 |
|------|------|------------|
| 基础配置 | 应用、UI、文件夹、领域、状态、类型、工作流 | 日常修改 |
| 高级配置 | 标签页、字段映射、数值限制、颜色、模板 | 很少修改 |

### 6.3 配置驱动

所有可变行为都通过配置控制：

| 行为 | 配置项 |
|------|--------|
| 文件夹位置 | `basic.folders.structure` |
| 状态列表 | `basic.statuses` |
| 类型列表 | `basic.types` |
| 领域和模块 | `basic.domains` |
| 工作流按钮 | `basic.workflows` |
| 标签页 | `advanced.tabs` |
| 字段名 | `advanced.fields` |
| 数值限制 | `advanced.limits` |
| 主题颜色 | `advanced.theme` |
| 周计划分类 | `advanced.templates` |
