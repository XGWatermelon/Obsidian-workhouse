# Obsidian 智能工作台插件 — 说明书

> **版本**：v2.1
> **更新日期**：2026-05-30
> **适用对象**：所有用户

---

## 一、插件简介

智能工作台是一个 Obsidian 插件，提供统一的工作台界面，集成计划管理、事项跟踪、学习分析、稍后阅读等功能。

### 核心特性

- **快速创建** — 一键创建事项（任务/想法/项目/写作/学习）和日志
- **计划管理** — 日计划、周计划，自动关联
- **状态监控** — 待办统计、逾期提醒、完成率追踪
- **学习分析** — 年度活跃度热力图、领域覆盖、学习状态分布
- **知识管理** — 事项清单、事项池、文档概览
- **稍后阅读** — 收藏摘录、待归类笔记管理
- **配置驱动** — 文件夹、状态、类型、领域全部可配置

---

## 二、安装

### 手动安装

1. 将以下文件复制到你的 Obsidian 仓库：
   ```
   <你的仓库>/.obsidian/plugins/worktop/
   ├── main.js
   ├── manifest.json
   └── styles.css
   ```
2. 打开 Obsidian → 设置 → 第三方插件 → 找到「智能工作台」→ 启用

### 从源码构建

```bash
git clone https://github.com/XGWatermelon/Obsidian-workhouse.git
cd Obsidian-workhouse
npm install
npm run build
# 产物：main.js, manifest.json, styles.css
```

## 三、快速开始

### 第一步：打开工作台

- 点击左侧边栏的布局图标
- 或按 `Ctrl+P` 输入「打开工作台」

### 第二步：创建今日日计划

- 按 `Ctrl+P` 输入「创建今日日计划」
- 或在工作台中点击「每日开始」按钮

如果本周计划不存在，会询问是否一起创建。创建后会自动在周计划中添加当日日计划的链接。

### 第三步：创建事项

- 按 `Ctrl+P` 输入「创建事项」
- 或在工作台顶部点击快捷创建按钮（任务/想法/项目/写作/学习）

弹窗中可设置：标题、状态、学习状态、截止日期。创建后文件会自动保存到对应类型的子文件夹中（如 `事项/任务/`）。

### 第四步：使用工作台

- **行动指北** — 查看今日要事、活跃任务、本周计划
- **知识管理** — 查看事项清单、事项池、最近笔记
- **学习分析** — 查看热力图、学习进度
- **稍后阅读** — 管理收藏摘录和待归类笔记

## 五、配置说明

### 配置文件位置

```
.obsidian/plugins/worktop/config.json
```

首次使用时自动创建默认配置。也可通过设置界面修改：设置 → 第三方插件 → 智能工作台 → 齿轮图标。

### 配置结构

配置分为 **基础配置**（日常使用）和 **高级配置**（一般不需要改）：

```json
{
  "version": 1,
  "basic": {
    "app": { "name": "工作台", "icon": "layout-dashboard", "viewTitle": "智能工作台" },
    "ui": { "openInMainEditor": true, "themeId": "green-dark" },
    "folders": { "root": "工作台", "structure": { ... } },
    "domains": [ ... ],
    "statuses": { "topic": [...], "learning": [...], "default": "...", "completed": [...] },
    "types": [ ... ],
    "workflows": [ ... ]
  },
  "advanced": {
    "tabs": [ ... ],
    "fields": { ... },
    "limits": { ... },
    "theme": { ... },
    "templates": { ... }
  }
}
```

### 基础配置详解

#### 应用配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `basic.app.name` | `工作台` | 应用名称 |
| `basic.app.icon` | `layout-grid` | 侧边栏图标（Lucide 图标名） |
| `basic.app.viewTitle` | `智能工作台` | 视图标题 |

#### UI 配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `basic.ui.openInMainEditor` | `true` | `true`=主编辑区打开，`false`=侧边栏打开 |
| `basic.ui.themeId` | `green-dark` | 主题 ID |

#### 文件夹配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `basic.folders.root` | `工作台` | 根目录名称 |
| `basic.folders.structure.dailyPlan.name` | `日计划` | 日计划文件夹 |
| `basic.folders.structure.dailyPlan.pattern` | `{date}-日计划` | 日计划文件名模式 |
| `basic.folders.structure.weeklyPlan.name` | `周计划` | 周计划文件夹 |
| `basic.folders.structure.weeklyPlan.pattern` | `{year}-W{week}-本周计划` | 周计划文件名模式 |
| `basic.folders.structure.diary.name` | `日记` | 日记文件夹 |
| `basic.folders.structure.inbox.name` | `工作台/待整理` | 待整理文件夹 |
| `basic.folders.structure.nodes.name` | `工作台/节点` | 知识图谱节点文件夹 |
| `basic.folders.structure.topic.name` | `事项` | 事项文件夹 |
| `basic.folders.structure.generatedDocs.name` | `清单` | 生成文档文件夹 |

**文件名模式变量**：
- `{date}` — 日期，如 `2026-05-29`
- `{year}` — 年份，如 `2026`
- `{week}` — 周数，如 `22`
- `{month}` — 月份，如 `05`
- `{day}` — 日期，如 `29`

**路径规则**：
- `name` 包含 `/` → 视为绝对路径，直接使用
- `name` 不包含 `/` → 拼接 `root`，如 `日计划` → `工作台/日计划`

#### 状态配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `basic.statuses.topic` | `["待评估", "进行中", "已完成", "已放弃"]` | 事项状态列表 |

| `basic.statuses.completed` | `["已完成", "已放弃"]` | 视为「已完成」的状态 |

## 六、工作流详解

### 6.1 创建日计划

**触发**：命令「创建今日日计划」/ 工作台「每日开始」按钮

**流程**：
1. 检查日计划文件夹，不存在则创建
2. 检查今日日计划是否已存在，存在则直接打开
3. 检查本周计划是否存在，不存在则询问是否一起创建
4. 创建日计划文件（含 frontmatter 和模板）
5. 在周计划中添加当日日计划的 wiki 链接

**生成内容**：
```markdown
---
tags:
  - 计划
  - 日计划
type: 日计划
date: 2026-05-29
weekly_plan: "[[工作台/周计划/2026-W22-本周计划.md]]"
---

# 2026-05-29 日计划

> 本周计划：[[工作台/周计划/2026-W22-本周计划.md]]

## 今日要事

- [ ] 任务 1
- [ ] 任务 2
- [ ] 任务 3

## 执行记录

### 上午
-
### 下午
-
### 晚上
-

## 完成情况

- 完成：
- 未完成：
- 原因：
```

### 6.2 创建周计划

**触发**：命令「创建本周计划」/ 创建日计划时自动询问

**生成内容**：

  - 周计划
type: 周计划
week: "2026-W22"
start_date: 2026-05-25

> 05.25 — 05.31

- [ ] 内容 1
- [ ] 内容 2

- [ ] 内容 1

### 写作/事项
- [ ] 内容 1

### 6.3 创建事项

**触发**：命令「创建事项」/ 工作台快捷创建按钮

**弹窗字段**：
- 标题（必填）
- 状态（下拉，从配置读取）
- 学习状态（下拉，从配置读取）
- 截止日期（日期选择器）

**注意**：创建的文件会自动保存到 `事项/{类型}/` 子文件夹中。例如选择「任务」类型，文件保存到 `事项/任务/` 目录下。

**生成内容**：
```markdown
---

type: 任务
status: 待评估
created: 2026-05-29
updated: 2026-05-29
deadline: ""
学习状态: 待阅读
---

# 事项标题

## 背景

## 目标

## 进展记录

### 2026-05-29
- 创建事项

## 参考资料

```

### 6.4 工作流按钮

点击工作流按钮会生成对应的统计文档，保存到 `清单/` 文件夹并自动打开：

| 按钮 | 生成文档 | 内容 |
|------|---------|------|
| 每日开始 | 打开/创建今日日计划 | 同创建日计划 |
| 任务总览 | `{date}-任务总览.md` | 所有进行中事项的表格 |
| 待整理清单 | `{date}-待整理清单.md` | 待整理文件夹中的文件列表 |
| 任务评估 | `{date}-任务评估.md` | 待评估事项的表格 |
| 写作管理 | `{date}-写作管理.md` | 写作类型事项的表格 |
| 学习统计 | `{date}-学习统计.md` | 各学习状态的统计表格 |
| 最近动态 | `{date}-最近动态.md` | 最近修改的文件列表 |

### 6.5 常用入口

**位置**：行动指北标签页 → 常用入口

**使用**：
- 点击「+」按钮 → 模糊搜索文件 → 添加到常用入口
- 点击已添加的链接 → 打开对应文档
- 点击「-」按钮 → 移除常用入口

**数据存储**：`工作台/常用入口.md` 文件，格式为 wiki 链接列表。

## 七、目录结构

### 推荐结构

```
用户的 Obsidian 仓库/
│
├── 工作台/                          ← 插件的地盘（自动生成）
│   ├── 日计划/                     ← 插件创建
│   ├── 周计划/                     ← 插件创建
│   ├── 事项/                       ← 插件创建（按类型分文件夹）
│   │   ├── 任务/
│   │   ├── 想法/
│   │   ├── 项目/
│   │   ├── 写作/
│   │   └── 学习/
│   ├── 节点/                       ← 插件创建（知识图谱节点）
│   ├── 待整理/                     ← 插件创建（收件箱）
│   ├── 清单/                       ← 插件创建（生成的统计文档）
│   └── 常用入口.md                 ← 插件创建
│
├── （用户自己的其他文件夹）          ← 用户管理，插件不碰
│
└── .obsidian/
```

### 设计原则

**插件只在工作台文件夹下生成内容，不篡改用户的其他文件夹。**

- 插件生成的文件/文件夹都在 `工作台/` 下
- 插件可以读取整个库的数据
- 插件不会移动、重命名用户的文件
- 用户可以随时把内容移进/移出工作台

### 用户迁移指南

如果用户已有 `节点/` 或 `待整理/` 文件夹：

**方案 A：移动内容到工作台下**
```
1. 把 用户/节点/ 下的内容移动到 工作台/节点/
2. 把 用户/待整理/ 下的内容移动到 工作台/待整理/
3. 删除空的原文件夹（可选）
```

**方案 B：修改配置指向用户文件夹**
```json
{
  "basic": {
    "folders": {
      "structure": {
        "nodes": { "name": "节点" },
        "inbox": { "name": "待整理" }
      }
    }
  }
}
```

## 九、属性下拉框

插件会自动在 Obsidian 中注册以下 frontmatter 字段的下拉选择：

| 字段 | 下拉选项来源 | 状态 |
|------|------------|------|
| `status` | `basic.statuses.topic` | 可用 |
| `学习状态` | `basic.statuses.learning` | 可用 |
| `type` | `basic.types` 的 name 列表 | 可用 |

### 使用方式

Obsidian 的属性选择框采用**清除后选择**的操作流程：

```
点击当前值 → 清空字段 → 出现下拉选项列表 → 选择新值
```

**详细步骤**：
1. 在文件的属性区域找到已注册的字段（如 `status`、`学习状态`、`type`）
2. **点击当前的属性值**（如 `待评估`）— 此时进入编辑状态
3. **清空输入框**（按 `Backspace` 或全选删除）— 清空后下拉选项列表自动出现
4. **从下拉列表中选择新值**（如 `进行中`）— 点击即可完成变更

**注意**：
- 如果不清空当前值，下拉选项列表不会出现
- 属性模式和源码模式都支持此操作
- 源码模式下输入字段名后也会自动弹出补全建议

> **状态说明**：属性下拉框功能已实现，需重启 Obsidian 后首次使用时生效。

## 十三、常见问题

### Q1: 首次打开工作台后，文件夹是怎么创建的？

**答**：插件采用「启动初始化+按需补充」策略 — 每次插件启动时检查所有必要的文件夹和文件，缺失的自动创建，已有的不会覆盖。禁用再启用插件即可触发补充。

### Q2: 为什么节点文件夹在工作台下面？

**答**：默认配置下，节点文件夹位于 `工作台/节点/`，与日计划、周计划等平级。这是为了遵守「插件只在工作台下生成」的设计原则。如果你希望节点在工作台外面，可以修改配置为 `{ "name": "节点" }`（绝对路径）。

### Q3: 如何自定义周计划的分类？

**答**：修改配置中的 `advanced.templates.weeklyCategories`：

```json
{
  "advanced": {
    "templates": {
      "weeklyCategories": ["我的分类1", "我的分类2", "我的分类3"]
    }
  }
}
```

### Q4: 如何修改事项状态列表？

**答**：修改 `basic.statuses.topic`，同时更新 `basic.statuses.default` 和 `basic.statuses.completed`：

```json
{
  "basic": {
    "statuses": {
      "topic": ["待办", "进行中", "完成", "取消"],
      "default": "待办",
      "completed": ["完成", "取消"]
    }
  }
}
```

### Q5: 如何添加新的领域和模块？

**答**：修改 `basic.domains`：

```json
{
  "basic": {
    "domains": [
      {
        "id": "frontend",
        "name": "前端",
        "color": "#3B82F6",
        "modules": [
          { "id": "react", "name": "React" },
          { "id": "vue", "name": "Vue" }
        ]
      }
    ]
  }
}
```

### Q6: 如何备份配置？

**答**：复制 `.obsidian/plugins/worktop/config.json` 文件即可。

### Q7: 有些文件/文件夹不知道从哪来的？

**答**：参见本文档第六节「工作流详解」，每个工作流按钮都会生成对应的统计文档到 `清单/` 文件夹。常用入口数据存储在 `工作台/常用入口.md`。

### Q8: 事项文件按什么规则存放在子文件夹中？

**答**：创建事项时选择的类型（任务/想法/项目/写作/学习）决定存放的子文件夹。例如创建一个「任务」类型的事项，会保存到 `事项/任务/` 文件夹下。插件查询时会自动搜索所有子文件夹，无需手动管理。
